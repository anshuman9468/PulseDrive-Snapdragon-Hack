API SPECIFICATION
================================================================================

ENDPOINTS
  POST /api/predict              - Generate health prediction
  GET /api/predict/health        - Check service health

REQUEST FORMAT (POST /api/predict)
  {
    "vehicleId": "VH123456",     # Required
    "features": {...},            # Optional
    "timestamp": "ISO 8601"       # Auto-generated if omitted
  }

RESPONSE FORMAT (200 OK)
  {
    "vehicleId": "VH123456",
    "healthScore": 85.5,          # 0-100
    "status": "healthy",          # healthy|warning|critical
    "confidence": 0.92,           # 0-1
    "recommendation": "...",      # Actionable text
    "timestamp": "ISO 8601"
  }

ERROR RESPONSES
  400: Invalid request (empty vehicleId)
  422: Validation error (missing required field)
  500: Prediction service error

GETTING STARTED
================================================================================

1. START SERVER
   cd backend
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

2. TEST WITH CURL
   curl -X POST "http://localhost:8000/api/predict" \
     -H "Content-Type: application/json" \
     -d '{"vehicleId": "VH001"}'

3. CHECK HEALTH
   curl "http://localhost:8000/api/predict/health"

4. VIEW API DOCUMENTATION
   http://localhost:8000/docs (Swagger UI)
   http://localhost:8000/redoc (ReDoc)

5. RUN EXAMPLES
   python example_prediction_usage.py
